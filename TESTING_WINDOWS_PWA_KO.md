# Windows PWA 수동 인수 테스트

테스트 전 JSON 전체 백업을 만드세요. 운영 Firestore에는 자동 테스트 데이터를 만들지 않으며, 수동 테스트 노트는 알아보기 쉬운 제목을 사용합니다. Android와 Windows는 같은 Firebase 프로젝트 및 허용된 Google 계정으로 로그인합니다.

## A. 인증과 개인 잠금

- [ ] Google 팝업 로그인이 성공한다.
- [ ] 팝업을 차단했을 때 redirect 로그인으로 진행할 수 있다.
- [ ] PWA를 완전히 닫고 다시 실행해도 로그인이 유지된다.
- [ ] 계정 이메일이 왼쪽 아래에 표시된다.
- [ ] 로그아웃 후 노트 리스너가 해제되고 Firestore 화면을 읽지 않는다.
- [ ] 다른 Google 계정은 `permission-denied`로 차단된다.
- [ ] 미인증 상태에서는 Firestore 네트워크 호출이 없다.

## B. Android → Windows

Android 앱을 실행한 상태에서 각각 변경하고 PWA에 자동 반영되는지 확인합니다.

- [ ] 새 노트
- [ ] 제목 및 빈 제목
- [ ] 여러 줄 본문과 긴 본문
- [ ] 체크리스트 추가·텍스트·체크/해제
- [ ] 체크리스트 순서
- [ ] 노트 삭제
- [ ] 7개 색상 프리셋
- [ ] `HEX 직접 입력` 노트가 웹에서 블랙 월넛 fallback과 제한 안내로 안전하게 보인다.

## C. Windows → Android

- [ ] `Ctrl+N`과 버튼으로 만든 새 노트가 Android에 나타난다.
- [ ] 입력 중에는 Firestore 쓰기가 발생하지 않고, 멈춘 뒤 500ms에 한 번만 쓴다.
- [ ] 제목·본문·체크리스트가 Android에 반영된다.
- [ ] 체크 항목의 위/아래 버튼과 `Alt+↑/↓` 순서가 유지된다.
- [ ] 노트 전환만으로 동일 데이터를 다시 쓰지 않는다.
- [ ] Android 위젯 설정에서 새 노트를 선택할 수 있다.
- [ ] 위젯의 제목, 본문, 체크리스트, 프리셋이 Room 반영 후 갱신된다.
- [ ] 10초 삭제 취소를 누르면 Android에서도 노트가 남아 있다.
- [ ] 취소하지 않으면 10초 뒤 양쪽에서 삭제된다.

## D. Android 앱 종료 상태

1. Android 앱을 최근 앱에서도 닫습니다.
2. Windows에서 기존 노트를 수정합니다.

- [ ] Android 위젯의 새로고침 버튼을 누르면 one-shot server pull 후 즉시 반영된다.
- [ ] 오프라인 또는 로그아웃 상태에서 새로고침해도 기존 위젯 내용이 사라지지 않는다.
- [ ] 네트워크가 연결되면 WorkManager가 약 1시간 주기로 확인한다.
- [ ] Android 배터리 최적화 때문에 실제 시각이 지연될 수 있음을 확인한다.
- [ ] 약 24시간 전체 ID 대조 또는 수동 새로고침에서 원격 삭제가 반영된다.
- [ ] `syncPending` 로컬 노트는 충돌 스냅샷 없이 조용히 유실되지 않는다.

## E. 오프라인

개인 PC 모드에서 온라인으로 한 번 전체 노트를 받은 뒤 테스트합니다.

- [ ] 네트워크를 끊어도 캐시된 노트를 열 수 있다.
- [ ] 기존 노트를 수정할 수 있다.
- [ ] 새 노트를 만들 수 있다.
- [ ] `오프라인` 또는 `오프라인 변경 보관됨` 상태가 표시된다.
- [ ] PWA를 닫았다 열어도 개인 PC 모드에서 캐시된 노트가 보인다.
- [ ] 네트워크를 복구하면 `연결 복구 중`을 거쳐 자동 업로드된다.
- [ ] Android 앱과 위젯에 변경이 반영된다.
- [ ] 공용 PC 모드에서는 브라우저 재시작 뒤 영구 캐시를 기대하지 않는다.

## F. 충돌

1. Android와 Windows에서 같은 노트를 엽니다.
2. Windows에서 입력한 뒤 500ms가 지나기 전에 Android에서 같은 노트를 저장합니다.

- [ ] Windows 초안이 갑자기 원격 내용으로 바뀌지 않는다.
- [ ] “다른 기기에서 이 노트가 변경되었습니다.” 배너가 나타난다.
- [ ] **원격 변경 불러오기** 전에 초안 유실 확인이 표시된다.
- [ ] 확인하면 원격 내용이 적용된다.
- [ ] **내 변경으로 덮어쓰기**를 선택하면 더 새로운 `updatedAt`으로 현재 초안이 저장된다.
- [ ] 자신의 pending write는 충돌 배너를 만들지 않는다.

## G. PWA 설치와 업데이트

- [ ] Firebase Hosting HTTPS 주소에서 Edge/Chrome 설치 아이콘이 나타난다.
- [ ] 설치 후 주소창 없는 독립 창으로 실행된다.
- [ ] 시작 메뉴와 Alt+Tab에 독립 앱으로 표시된다.
- [ ] 작업표시줄 고정이 가능하다.
- [ ] 약 430×720과 최소 360×520에서 목록/편집 화면 전환이 가능하다.
- [ ] Windows 125%, 150% 배율에서 버튼과 입력이 겹치지 않는다.
- [ ] 새 서비스 워커가 대기하면 **업데이트 가능** 안내가 나타난다.
- [ ] **저장 후 적용**이 대기 초안을 먼저 flush한다.
- [ ] 업데이트 뒤 마지막 노트와 로컬 초안이 보존된다.

## H. 백업과 복원

- [ ] JSON 백업에 포맷 버전, 생성 시각, 허용된 8개 필드만 있다.
- [ ] Markdown 파일에 제목, 본문 줄바꿈, 체크 상태가 보존된다.
- [ ] 복원 전 유효·무효 개수와 오류가 표시된다.
- [ ] 건너뛰기 전략이 기존 noteId를 쓰지 않는다.
- [ ] 최신 전략이 더 큰 `updatedAt`만 쓴다.
- [ ] 복사 전략이 새 UUID를 만든다.
- [ ] 예상 Firestore 쓰기 횟수가 최종 확인 전에 표시된다.
- [ ] 허용되지 않은 필드는 제거되고 Firestore에 기록되지 않는다.

## I. 크기와 안전한 렌더링

- [ ] 약 800KiB 이상 UTF-8 노트에서 분할 권고가 표시된다.
- [ ] 약 950KiB 이상은 저장이 차단되며 편집 내용이 남는다.
- [ ] 제목·본문·체크리스트에 `<script>` 같은 문자열을 넣어도 HTML로 실행되지 않고 일반 텍스트로 보인다.
- [ ] 외부 CDN, 분석, 광고, 오류 수집 요청이 없다.

## J. 무료 사용량과 리스너

개발 모드 오른쪽 아래 카운터와 브라우저 Network 탭을 함께 확인합니다.

- [ ] 활성 컬렉션 리스너 `L`은 정확히 1개다.
- [ ] 선택 노트 변경으로 새 listener가 생기지 않는다.
- [ ] 검색으로 Firestore 호출이 생기지 않는다.
- [ ] 입력 키마다 쓰지 않는다.
- [ ] 입력 정지 후 문서 하나에 쓰기 1회다.
- [ ] metadata-only snapshot이 다시 쓰기를 만들지 않는다.
- [ ] 짧은 polling이 없다.
- [ ] Android WorkManager는 고유 이름으로 1개만 등록된다.

## 자동 검증 명령

```powershell
cd web
npm ci
npm run lint
npm run test
npm run build
npm audit --audit-level=high
cd ..
./gradlew testDebugUnitTest
./gradlew assembleDebug
```

Firebase Emulator를 연결하는 통합 테스트는 선택사항입니다. 운영 프로젝트를 자동 테스트 대상으로 사용하지 않습니다.
